## 実施内容

### 1. 現状分析

**問題点:**
- フィードバックループの説明が長く、実施タイミングが不明確
- 作業記録と改善記録の違いがわかりにくい
- タスク実行フローが複雑で、ステップが多すぎる

**改善方針:**
- タスク実行フローを3段階に簡素化（開始→実行→完了）
- フィードバックループを「タスク完了後の振り返り」として明確化
- 作業記録と改善記録の違いを表形式で整理

### 2. ドキュメント改善

**変更内容:**
- タスク実行フローを簡素化
- フィードバックループの説明を「振り返りと改善」セクションに統合
- 作業記録と改善記録の違いを明確化
- Git Commit & Pushのタイミングを明確化

### 3. 改善内容の詳細

**主な変更点:**

1. **タスク実行フローの簡素化**
   - 3ステップに整理（開始→実行→完了）
   - 各ステップで何をすべきかを明確化
   - 絵文字を使って視覚的にわかりやすく

2. **フィードバックループの明確化**
   - 「振り返りと改善」セクションに統合
   - 「いつ振り返るか？」を明示（問題がなければ不要）
   - 改善記録の作成タイミングを明確化

3. **作業記録と改善記録の違いを明確化**
   - 表形式で比較
   - 作成タイミング、目的、内容、対象を整理

4. **Git Commit & Pushの簡素化**
   - コマンド例を簡潔に
   - コミットメッセージの例を簡略化
